# Feature Backlog

Feature ideas that are worth preserving, but are not yet committed product scope.

## URL-Backed Table State

Encode the current table view state in the URL so every view can be shared or reopened directly.

State to encode:

- Model
- Page
- Page size
- Sort
- Search
- Filters
- Selected row
- Visible columns

Notes:

- The URL should be the source of truth for shareable table state.
- Shared links should restore the same model, table configuration, and selected record when possible.
- Invalid or stale URL values should degrade gracefully, such as falling back to the default model or clearing an unavailable selected row.

## Column Picker

Let users customize table columns per model.

Column settings:

- Hide columns
- Reorder columns
- Pin columns
- Resize columns

Persistence:

- Store preferences per model in local storage.
- Restored settings should apply automatically when returning to a model.
- Missing or renamed fields should be ignored gracefully.

## Better Filters

Expand table filtering beyond simple search so users can express common Prisma-compatible predicates.

Filter operators and controls:

- `gt`
- `gte`
- `lt`
- `lte`
- `between`
- `in`
- `not in`
- Date ranges
- Boolean dropdowns
- Null and not-null shortcuts

Backend fit:

- Prisma Client supports structured `where`, `orderBy`, pagination, and relation querying, so these filters should map naturally to the existing backend boundary.
- Filter requests should remain structured rather than exposing arbitrary query execution.

## Aggregation Panel

Show lightweight summary statistics for the selected model.

Model metrics:

- Total row count

Field metrics:

- Null counts
- Enum value distribution
- Minimum and maximum dates
- Numeric minimum, maximum, and average

Notes:

- Aggregations should be scoped to the selected model.
- Expensive metrics may need lazy loading, caching, or an explicit refresh action for large tables.

## Relation-Aware Navigation

Let users navigate between related records directly from table cells and the record preview.

Navigation behavior:

- Make foreign-key-like fields such as `userId` and `postId` clickable when they map to a known relation.
- Jump to the related model and select the related record.
- Preserve enough navigation state for users to move back to the previous view.

Preview enhancements:

- Show relation counts in the record preview.
- Support common relation shapes, including one-to-one, one-to-many, and many-to-many relations.

Backend fit:

- Prisma relations cover one-to-one, one-to-many, and many-to-many patterns, so this feature should map naturally to Prisma metadata and relation querying.
- Relation traversal should stay bounded and explicit to avoid unexpectedly expensive queries.

## Multi-Env Switcher

Let users switch which environment file powers the viewer.

Environment options:

- `.env.local`
- `.env`
- `.env.preview`
- `.env.production-readonly`
- Other discovered `.env.*` files

Safety requirements:

- Show clear labels for the currently selected environment.
- Make production-like environments visually distinct and harder to select accidentally.
- Prefer read-only database credentials for any non-local environment.
- Require an explicit reconnect or restart behavior if changing environments cannot be handled safely at runtime.

## Query Inspector

Show the Prisma query object generated by the current table view.

Query fields to inspect:

- `where`
- `orderBy`
- `select`
- `skip`
- `take`

Use cases:

- Help users debug why a table view returns specific rows.
- Help users learn how viewer controls map to Prisma Client query options.
- Provide a copyable structured representation without allowing arbitrary query execution from the browser.
